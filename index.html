HTML loob struktuuri (skeleti), CSS annab välimuse. Standardid tagavad, et kood töötab kõikjal. Veebileht koosneb alati kahest suuremast osast: päisest (<head>) ja sisust (<body>). Kõige ülal on deklaratsioon <!doctype html>, mis ütleb brauserile, et dokument järgib HTML5 standardit. See väldib nn quirks-mode’i, kus brauserid püüavad vana koodi "arvata". < lang="et"> määrab dokumendi alguse ja keele, mis aitab otsimootoritel ja ekraanilugejatel sisu mõista. Päisesse (<head> käib: <meta charset="utf-8"> – määrab, et tekst on UTF-8 kodeeringus (õiged täpitähed). <meta name="viewport" content="width=device-width, initial-scale=1"> – seab mobiilse seadme vaate laiuse ja skaala õigeks. <title> lehe nimi, mis kuvatakse brauseri sakil. <link rel="stylesheet" href="assets/css/style.css"> viide välisele CSS-failile. Miks peab style kaust linkima? Kui stiilid on eraldi failis, siis: Kood on puhtam sisaldab vaid sisu ja struktuuri, mitte kujundust. Taaskasutus sama CSS fail saab kujundada mitut lehte. Jõudlus brauserid saavad CSS-faili vahemällu salvestada ja kiiremini laadida. Hooldatavus stiile saab muuta ühes kohas ja need rakenduvad kõikjal. Samm-sammult. Ava VS Code, loo index. ja css kaust, mille sisse style.css. Lisa DOCTYPE ja põhistruktuur. Seo CSS <link> abil. Lisa skip-link, et klaviatuurikasutaja saaks kiiresti põhisisusse liikuda.
